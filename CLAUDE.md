## コミットメッセージのルール

コミットメッセージは必ず以下の形式で書くこと：

```
【課題】何の問題を解決したか
【解決】どう解決したか
【指示】ユーザーからのプロンプトイメージ（抽象的な言葉）→Claudeが具体化した内容
【手作業】ユーザーが自分で行った手動作業（なければ「なし」）
```

### 具体例

```
【課題】ツイート内容が技術的すぎて伝わらない【解決】Gemini APIで口語調に自動変換【指示】「ツイートがわかりにくい」→Geminiプロンプトを口語調に変更し自動変換の仕組みを実装【手作業】Gemini APIキー取得・GitHub Secretsに登録
```

### ルール

- `【】`の中の改行は不可。各項目は1行で書くこと
- 4つのタグがすべて存在する場合のみNotionに自動蓄積される（手作業は「なし」でも可）
- 【指示】は「ユーザーの言葉（抽象）→Claudeが具体化した内容」の形式で書くこと
- この形式に従うことで、GitHub ActionsがNotionに自動蓄積し、毎日20:00 JSTにXへ自動投稿される
